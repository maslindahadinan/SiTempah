3. Latar Belakang dan Pernyataan Masalah
Pada masa ini, proses tempahan fasiliti gunasama di JTM seperti bilik mesyuarat, dewan, makmal komputer dan
bilik seminar dijalankan secara manual melalui borang fizikal, e-mel atau panggilan telefon kepada kerani/pengurus
fasiliti. Pendekatan ini menimbulkan beberapa isu operasi:
(cid:127) (cid:127) Pertindihan tempahan (double booking) akibat ketiadaan paparan kekosongan fasiliti secara masa nyata
(real-time).
(cid:127) (cid:127) Proses kelulusan yang perlahan dan tidak telus kerana bergantung kepada komunikasi tidak berstruktur
(e-mel/WhatsApp).
(cid:127) (cid:127) Sukar untuk pemohon menyemak status permohonan tanpa perlu bertanya secara manual kepada pengurus.
(cid:127) (cid:127) Ketiadaan rekod berpusat (centralised record) bagi tujuan audit, pelaporan dan analisis penggunaan fasiliti.
(cid:127) (cid:127) Pengalaman pengguna (UX) borang sedia ada yang lapuk dan tidak mesra pengguna.
Justeru, JTM bercadang membangunkan sebuah sistem web bersepadu — SiTempah — yang membolehkan
tempahan fasiliti gunasama dilakukan secara dalam talian (online), dengan aliran kerja kelulusan berstruktur
(structured approval workflow) serta paparan status tempahan secara telus kepada semua pihak berkepentingan.

9. Aliran Status Permohonan (Booking Status Workflow)
Setiap permohonan tempahan akan melalui aliran status berikut. Reka bentuk status ini menyokong keperluan
Pengurus untuk meluluskan, menangguhkan (KIV) atau menolak sesuatu permohonan.
Status Penerangan Pencetus / Siapa Boleh
Ubah
Draf (Draft) Permohonan disimpan sementara sebelum dihantar rasmi. Pengguna
Menunggu Kelulusan Permohonan telah dihantar dan menunggu tindakan Sistem (auto) selepas
(Pending) Pengurus. penghantaran
KIV (Kept-In-View) Permohonan ditangguhkan buat sementara — memerlukan Pengurus
maklumat tambahan atau menunggu pengesahan lanjut.
Diluluskan (Approved) Permohonan diluluskan; fasiliti disahkan ditempah untuk slot Pengurus
berkenaan.
Ditolak (Rejected) Permohonan ditolak berserta sebab/catatan penolakan. Pengurus
Dibatalkan (Cancelled) Permohonan dibatalkan oleh pemohon sendiri sebelum tarikh Pengguna / Pentadbir
acara.
Tamat Tempoh (Expired) Permohonan automatik ditamatkan jika tiada tindakan Sistem (auto)
Pengurus selepas tempoh SLA ditetapkan.
Rajah Aliran Status (Ringkasan Teks)
Draf fi Menunggu Kelulusan fi { Diluluskan | KIV fi (Diluluskan | Ditolak) | Ditolak } ; Menunggu Kelulusan / KIV fi
Tamat Tempoh (jika SLA terlampau) ; Diluluskan fi Dibatalkan (atas permintaan pemohon).

& Tech Stack)
12.1 Gambaran Seni Bina
SiTempah dibangunkan menggunakan seni bina Jamstack moden yang ringan dan mudah diselenggara, sesuai
untuk pasukan kecil yang menggunakan bantuan AI coding assistant (Z.AI — model GLM 5.2) dalam proses
pembangunan.
Lapisan Teknologi / Perkhidmatan Fungsi
Antara Muka React / Next.js (atau Vite + React) dengan Paparan UI Glassmorphism, kalendar interaktif,
(Frontend) TailwindCSS, dijana/dibantu oleh Z.AI borang tempahan, papan pemuka.
GLM 5.2
Backend-as-a-Servi Supabase (PostgreSQL, Supabase Auth, Pangkalan data, pengesahan pengguna, storan fail
ce Supabase Storage, Realtime lampiran, dan kemas kini masa nyata (status
Subscriptions, Row Level Security) tempahan).
Logik Sisi Pelayan Supabase Edge Functions / Netlify Logik perniagaan kompleks: semakan konflik
(Backend Logic) Functions (Node.js/TypeScript) tempahan, penghantaran notifikasi e-mel,
penjanaan laporan.
Pengesahan Supabase Auth (E-mel & Kata Laluan / Log masuk selamat, pengurusan sesi (JWT), dan
Pengguna (Auth) Magic Link) integrasi RBAC melalui custom claims/roles table.
Notifikasi Supabase Edge Functions + Menghantar e-mel automatik apabila status
perkhidmatan e-mel (contoh: permohonan berubah.
Resend/SendGrid)
Penempatan Netlify (Continuous Deployment daripada Hosting antara muka web, CDN global, HTTPS
(Hosting) Git repository) automatik, pratonton pembangunan (deploy
previews).
Kawalan Versi Git (GitHub/GitLab) Pengurusan kod sumber dan integrasi CI/CD
dengan Netlify.
Pembantu Z.AI — Model GLM 5.2 Menjana dan membantu menulis kod
Pembangunan AI frontend/backend, komponen UI Glassmorphism,
dan skrip migrasi Supabase.
12.2 Rajah Seni Bina Tahap Tinggi (Naratif)
Pengguna berinteraksi dengan aplikasi web (React/Next.js) yang dihoskan di Netlify. Aplikasi ini berkomunikasi
secara terus dengan Supabase melalui Supabase Client SDK bagi operasi pengesahan (auth), pertanyaan
pangkalan data (query melalui PostgREST API automatik), storan fail, dan langganan masa nyata (realtime) bagi
kemas kini status tempahan. Logik perniagaan yang lebih kompleks (contoh: pengesahan konflik tempahan sebelum
simpan, penghantaran notifikasi e-mel, penjanaan laporan PDF) dikendalikan melalui Supabase Edge Functions
atau Netlify Functions. Semua akses data dikawal ketat melalui dasar Row Level Security (RLS) berdasarkan
peranan pengguna yang disahkan melalui token JWT daripada Supabase Auth.

14. Reka Bentuk UI/UX — Glassmorphism Design System
Antara muka SiTempah akan menggunakan gaya reka bentuk Glassmorphism — panel separa lut sinar
(translucent) dengan kesan kabur latar belakang (backdrop blur), sempadan halus (subtle border) dan bayang
lembut (soft shadow) bagi mewujudkan kesan 'kaca' yang moden, ringan dan mendalam (depth).
14.1 Prinsip Reka Bentuk Utama
(cid:127) (cid:127) Kesan Kaca (Glass Panels): Kad (card), modal dan bar navigasi menggunakan latar belakang separa lut sinar
(opacity 10–20%) dengan `backdrop-filter: blur(16–24px)`.
(cid:127) (cid:127) Latar Belakang Gradien Dinamik: Gradien lembut warna korporat JTM (biru gelap ke teal) sebagai latar untuk
menonjolkan panel kaca.
(cid:127) (cid:127) Sempadan & Bayang Halus: Sempadan 1px putih lut sinar (opacity ~20–30%) dan bayang lembut (soft
box-shadow) bagi memberi kesan timbul (elevation) tanpa berat.
(cid:127) (cid:127) Palet Warna: Warna primer Biru Kementerian (#0F4C81), aksen Teal (#14B8A6), dengan status berwarna:
Hijau (Diluluskan), Kuning/Amber (KIV), Merah (Ditolak), Kelabu (Menunggu/Draf).
(cid:127) (cid:127) Tipografi: Fon sans-serif moden (cth: Inter/Poppins) dengan hierarki jelas — tajuk tebal, teks badan mudah
dibaca dengan kontras mencukupi di atas latar kaca.
(cid:127) (cid:127) Mikro-interaksi: Animasi peralihan lembut (hover, klik, transisi kad) bagi memberi maklum balas visual tanpa
mengganggu prestasi.
(cid:127) (cid:127) Responsif Sepenuhnya: Susun atur menyesuaikan diri (adaptive layout) untuk paparan desktop, tablet dan
mudah alih, dengan navigasi mudah alih berasaskan menu bawah/hamburger.
(cid:127) (cid:127) Aksesibiliti: Walaupun menggunakan kesan lut sinar, kontras teks-ke-latar dikekalkan pada tahap boleh baca
(mematuhi garis panduan WCAG AA asas).

sinar untuk mewujudkan rupa seperti kaca.

statik/serverless.
